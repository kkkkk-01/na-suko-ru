1

Docker Desktop をインストール

Dockerはコンテナを動かすためのソフトです（無料）

1-1. Docker公式サイトにアクセス

以下のリンクをクリックしてダウンロードページを開きます：

Docker Desktop ダウンロードページ

この画面が表示されます。「Download for Windows」をクリック

1-2. インストーラーを実行

  1. a ダウンロードした Docker Desktop Installer.exe をダブルクリック
  2. b 「Use WSL 2 instead of Hyper-V」に チェックが入っていることを確認
  3. c 「OK」をクリックしてインストール開始（3〜5分かかります）

インストール画面。WSL 2のチェックが入っていることを確認

1-3. インストール完了 → PCを再起動

重要：PCを再起動してください

インストール完了後、必ずPCを再起動してください。再起動しないとDockerが正しく動作しません。

2

WSL 2 を有効にする

DockerがLinuxコンテナを動かすために必要です

Windows 11 や最新のWindows 10では、Docker Desktopが自動でWSL 2をセットアップしてくれる場合があります。Step 3 でDockerが正常起動すれば、この手順はスキップしてOKです。

2-1. PowerShellを「管理者として実行」で開く

  1. a スタートボタンを 右クリック
  2. b 「Windows PowerShell（管理者）」または「ターミナル（管理者）」をクリック
  3. c 「このアプリがデバイスに変更を加えることを許可しますか？」→「はい」

スタートボタン右クリック → 管理者として実行

2-2. WSLインストールコマンドを実行

以下のコマンドをコピーして、PowerShellに貼り付けてEnterを押してください：

wsl --install

完了したらもう一度PCを再起動

WSLのインストール後、再度PCを再起動してください。

3

Docker Desktop を起動・確認

Dockerが正しく動くか確認します

3-1. Docker Desktop を起動

  1. a デスクトップの Docker Desktop アイコンをダブルクリック
  2. b 画面右下のタスクバーに クジラアイコン が表示されるのを待つ
  3. c クジラアイコンのアニメーションが止まったら起動完了（1〜2分かかります）

Docker Desktop が起動した状態。左下が「Engine running」になればOK

3-2. Dockerの動作確認

PowerShell（通常モードでOK）を開いて、以下を実行：

docker --version

こう表示されればOK！

Docker version 27.x.x, build xxxxxxx

もう一つ確認：

docker compose version

こう表示されればOK！

Docker Compose version v2.x.x
4

プロジェクトファイルを展開

ダウンロードしたファイルをPCに展開します

4-1. ファイルをダウンロード

以下のボタンからプロジェクトファイルをダウンロードしてください：

nursecall-system.tar.gz をダウンロード (64KB)

4-2. PowerShellで展開する

PowerShellを開いて、以下のコマンドを 1行ずつ 実行してください：

ダウンロードフォルダに移動：

cd ~\Downloads

展開先フォルダを作成：

mkdir nursecall-system

ファイルを展開：

tar -xzf nursecall-system.tar.gz -C nursecall-system

プロジェクトフォルダに移動：

cd nursecall-system

確認コマンド

dir docker-compose.yml を実行して、ファイルが表示されればOK！

5

環境設定ファイルを作成

1コマンドだけ！設定はデフォルトでOK

5-1. .envファイルを作成

以下のコマンドを実行するだけです：

copy .env.example .env

ファイルの中身を変更する必要はありません。Phase1（技術検証版）ではデフォルト値のままで全機能が動きます。

6

ビルド＆起動！ （ここがメイン！）

いよいよシステム全体を起動します

6-1. Docker イメージをビルド（初回のみ）

以下のコマンドを実行してください。初回は5〜15分かかります（回線速度によります）：

docker compose build

よくあるトラブルと解決策

「port is already allocated」エラー

他のアプリケーションがポートを使用中です。

解決策：

  1. 使用中のアプリを終了する（例：他のWebサーバー、Skypeなど）
  2. または、PowerShellで競合プロセスを確認： netstat -ano | findstr :3000
「Cannot connect to the Docker daemon」エラー

Docker Desktopが起動していません。

解決策：

Docker Desktopアプリを起動して、タスクバーのクジラアイコンが安定するまで待ってから再度コマンドを実行してください。

コンテナが「Restarting」を繰り返す

データベース起動に時間がかかっている可能性が高いです。

解決策：

  1. 30秒〜1分待つ
  2. docker compose ps で再確認
  3. それでも直らない場合：docker compose logs api でエラーを確認
Windowsファイアウォールの警告が出た

Dockerが通信するための許可を求めています。

解決策：

「プライベートネットワーク」と「パブリックネットワーク」の両方にチェックを入れて「アクセスを許可する」をクリックしてください。

スマートフォンからアクセスしたい

同じWi-Fiに接続していれば、PCのIPアドレスでアクセスできます。

IPアドレスの確認：

ipconfig | findstr /i "IPv4"

例：IPが 192.168.1.100 の場合 → http://192.168.1.100:3002
